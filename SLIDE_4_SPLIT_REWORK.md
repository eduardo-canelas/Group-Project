# Distribution of work slide rework

This replaces the current crowded "Distribution of Work" slide with two cleaner slides.

Use the same visual theme you already have:

- dark navy background
- soft cream corner shapes
- gold title text
- light blue content cards
- blue label tabs
- black body text

## Why this change helps

Right now the slide is trying to do two different jobs at the same time:

1. show who worked on what
2. show the tech stack

That makes the slide feel crowded and harder to explain.

Splitting it into two slides makes it easier for the audience to read and easier for the speaker to present.

---

## New slide 4
## Title: Distribution of Work

### Layout

Use two main columns instead of three.

Left card:

- label tab: `WHO`
- list of team members

Right card:

- label tab: `WHAT`
- matching responsibilities

Keep the arrows if you want, but the slide will look cleaner even without them.

The main goal is to make the relationship easy to read in one quick look.

### Exact slide text

#### Left card label

`WHO`

#### Left card content

- Guiscard Marcelin
- Liliana Flores
- Avery Brewer
- Ashton Moulton
- Eduardo Canelas

#### Right card label

`WHAT`

#### Right card content

- React Integration
- Styling / Planning
- Styling / Planning
- React Integration
- Backend Programming

### Better speaking flow for this slide

"This slide shows how the work was divided across the team.

Guiscard worked on React integration.

Liliana and Avery focused on styling and planning.

Ashton also worked on React integration.

Eduardo handled the backend programming.

Splitting the work this way helped us build the front end, backend, and design in parallel while still connecting everything through the same app."

### Design notes

- Make the two cards slightly wider than the old three-card layout.
- Increase line spacing so each person and task has room to breathe.
- Keep names and tasks centered vertically.
- If you keep arrows, make them thinner and shorter so they do not crowd the text.
- If space allows, bold just the names or just the jobs, not both.

## New slide 5
## Title: Technology Stack

### Layout

Use one centered card or two smaller cards.

Best option:

- one centered card with a `TECH STACK` label
- technologies listed in a clean vertical stack

This slide should feel simpler than the work-distribution slide because it is only communicating one idea.

### Exact slide text

#### Card label

`TECH STACK`

#### Card content

- MERN Stack
- MongoDB Atlas
- Express.js
- React
- Node.js
- Tailwind CSS
- Axios
- React Router

### Optional grouped version

If you want the slide to feel more organized, split the stack into two groups:

#### Frontend

- React
- Tailwind CSS
- Axios
- React Router

#### Backend / Database

- Node.js
- Express.js
- MongoDB Atlas
- Mongoose

This grouped version is actually stronger because it explains what each tool did instead of showing one long list.

### Best speaking flow for this slide

"This slide shows the technologies we used to build the project.

On the front end, we used React for the user interface, React Router for page navigation, Axios for API requests, and Tailwind CSS for styling.

On the back end, we used Node.js and Express.js for the server logic.

For the database, we used MongoDB Atlas, and we used Mongoose to define our data models and work with the database more easily.

Together, these tools gave us a full MERN application with a styled front end, protected routes, and a structured database."

### Design notes

- If you use one card, keep the list centered and evenly spaced.
- If you use two cards, label them `FRONTEND` and `BACKEND / DATABASE`.
- Keep the font size a little larger than the current tech list so it is easier to read from far away.
- Add `Tailwind CSS`, `Axios`, `React Router`, and `Mongoose` if you want the slide to match the code more accurately.

## Best final recommendation

Use this structure:

### Slide 4

`Distribution of Work`
